# Classical and Neural Models for Sentiment Analysis

This repository contains a multiclass sentiment analysis project based on the **TweetEval sentiment dataset**. The objective is to compare a classical machine learning baseline, two neural networks trained from scratch, and a pretrained transformer under the same experimental protocol.

Repository: https://github.com/JuanMiguelPinos/Classical_Neural_Models_Sentiment_Analysis

## Models

The project compares four approaches:

- **TF-IDF + LinearSVC** using word and character n-grams.
- **TextCNN** with trainable word embeddings and convolutional filters.
- **BiGRU with Attention** using bidirectional recurrent layers and masked attention.
- **BERTweet** fine-tuned for three-class sentiment classification.

The labels are:

- `0`: negative
- `1`: neutral
- `2`: positive

## Dataset

The notebook downloads the official TweetEval sentiment splits automatically.

| Split | Rows |
|---|---:|
| Train | 45,615 |
| Validation | 2,000 |
| Test | 12,284 |

The official splits are kept unchanged. The test set has a different class distribution from training and validation, which makes macro-F1 especially useful for evaluation.

## Experimental protocol

- TextCNN, BiGRU, and BERTweet are trained with seeds `13`, `42`, and `77`.
- The final neural result is a soft-voting ensemble obtained by averaging the probabilities of the three runs.
- LinearSVC hyperparameters are selected with the official validation split.
- Neural checkpoints are selected using validation macro-F1 and early stopping.
- The main metric is macro-F1.
- Accuracy, macro precision, macro recall, weighted F1, class-level F1, bootstrap confidence intervals, and McNemar tests are also calculated.

## Final results

| System | Accuracy | Macro-F1 |
|---|---:|---:|
| TF-IDF + LinearSVC | 0.6121 | 0.6079 |
| TextCNN ensemble | 0.5991 | 0.5894 |
| BiGRU ensemble | 0.6083 | 0.5996 |
| BERTweet ensemble | **0.7178** | **0.7184** |

BERTweet achieved the best result and was also the most stable neural model across random seeds. LinearSVC remained competitive with the two networks trained from scratch while requiring much less training time, storage, and inference cost.

## Running the project in Google Colab

Google Colab with an NVIDIA T4 GPU is the recommended environment.

1. Open `sentiment_analysis_project.ipynb` in Google Colab.
2. Select **Runtime > Change runtime type > T4 GPU**.
3. Run all cells from the beginning.
4. The notebook downloads TweetEval and BERTweet automatically.
5. At the end, it generates and downloads:

```text
/content/sentiment_project_results.zip
```

A complete execution normally takes around one hour on a T4 GPU. Most of the time is used by the three BERTweet fine-tuning runs.

## Local installation

Create and activate a virtual environment, then install the dependencies:

```bash
python -m venv .venv
```

Linux or macOS:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

Windows:

```powershell
.venv\Scripts\activate
pip install -r requirements.txt
```

The notebook is the main executable version. The `.py` file is included as a readable source export. Depending on how it was exported from Colab, notebook-specific commands such as `%pip` may need to be removed or replaced before running it directly as a normal Python script.

## Main conclusions

- Domain-specific pretraining provides the clearest improvement on TweetEval.
- BERTweet is both the strongest and the most stable model.
- A more complex neural architecture does not automatically outperform a classical baseline when trained from scratch on a moderate-sized labeled dataset.
- LinearSVC is the best lightweight option when speed, storage, and simplicity are important.
- Ensembling improves all three neural approaches, especially TextCNN and BiGRU.
